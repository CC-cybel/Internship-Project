# 报告生成 (Report Generation)

## 技能描述
此技能用于生成 LeadBench 医疗对话评估的对比分析报告，通常在多个模型完成评估后，需要对它们的表现进行横向对比时使用。生成的报告包含核心指标对比、详细的数据分析以及针对模型优缺点的深度解读。

## 适用场景
- 当有新的模型评估结果产出，需要与基准模型（如 Claude-Sonnet-4.5）或其他版本模型进行对比时。
- 需要向团队展示模型迭代进展，特别是“硬性通过率”和“完美率”的变化时。
- 需要分析模型在不同维度（合规性、逻辑性、专业性等）的具体表现时。

## 输入要求
- **评估结果文件路径**：需要提供待对比模型的 `evaluation_report.md` 文件路径。
  - 例如：`/data1/yezj/gitlab/leadbench/output/model_name_version_timestamp/evaluation_report.md`
- **基准模型报告路径**（可选但推荐）：通常为 SOTA 模型（如 Claude）的评估报告，用于确立对比基准。

## 输出格式
输出为一个 Markdown 格式的汇总报告（例如 `comparison_summary.md`），结构如下：

### 1. 评估概述 (Overview)
- **评估模型**：列出所有参与对比的模型名称及版本。
- **评估数据集**：简要描述评估数据集来源及规模（例如：LeadBench 医疗对话评估集，共 339 条样本）。
- **评估规则体系**：简述 5 个核心维度（Compliance, Logic, Expertise, Pacing, Ai-ism）。
- **裁判模型**：说明使用的 Judge Model 配置（如 Qwen3.5-397B-A17B + Thinking）。

### 2. 评估结果 (Results)
- **核心指标对比表**：
  | 模型 | Average Score (平均分) | Hard Pass Rate (硬性通过率) | Sample Pass Rate (完美率) | Rule Macro Pass Rate (平均规则通过率) |
  | :--- | :--- | :--- | :--- | :--- |
  | Model A | 95.99 | **67.55%** | 42.48% | 87.05% |
  | Model B | 97.12 | 64.90% | **45.43%** | **91.88%** |

  > **注意**：
  > - **Average Score**：综合体感分数。
  > - **Hard Pass Rate**：**核心可用性指标**。未触发任何重罚分规则（扣分 > 1分）的样本比例。
  > - **Sample Pass Rate**：零容忍的成品率（得分为 100 分的样本比例）。
  > - **Rule Macro Pass Rate**：所有触发规则的平均通过率。
  > - 表格中最高值需 **加粗** 显示。

- **可视化图表**（可选）：如果有生成的对比图表，在此处插入。

### 3. 深度对比与分析 (Deep Dive)
- **Hard Pass Rate 分析**：重点对比“硬性通过率”，解释为什么该指标能更好地区分模型可用性（例如：Stage2 在避免严重错误上优于 Claude）。
- **短板与机会 (Gap Analysis)**：
  - 识别导致“完美率”低的主要原因（如：身份识别错误、机械化填充词、过度自信）。
  - 列出具体的低分规则 ID 和名称（如 Rule 32, Rule 34）。
  - 提出改进建议（Action Items），如数据清洗、Prompt 优化、RLHF 惩罚等。

### 4. 总结 (Summary)
- 一句话总结核心结论（例如：Stage2 在硬性指标上反超 SOTA，但在软体验上仍有提升空间）。
- 明确下一步迭代重点。

## 关键指标计算逻辑
- **Average Score**: $\frac{\sum \text{final\_score}}{\text{Total Samples}}$
- **Sample Pass Rate**: $\frac{\text{Count}(\text{final\_score} == 100)}{\text{Total Samples}}$
- **Hard Pass Rate**: $\frac{\text{Count}(\text{No Rule Deduction} > 1)}{\text{Total Samples}}$
- **Rule Macro Pass Rate**: $\text{Average of individual rule pass rates}$

## 操作步骤
1. **收集数据**：读取所有待对比模型的 `evaluation_report.md`，提取上述四个核心指标。
2. **生成表格**：构建 Markdown 表格，填入数据并标注最高值。
3. **撰写分析**：
   - 比较 Hard Pass Rate，强调模型在“安全性/合规性”上的表现。
   - 比较 Sample Pass Rate，指出模型在“拟人化/完美体验”上的差距。
   - 检查低分规则，归纳共性问题（如“AI味重”、“喜欢反问”）。
4. **输出文件**：将内容写入新的 Markdown 文件。
