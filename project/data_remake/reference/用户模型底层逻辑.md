# 用户模型底层逻辑

# 一、人格模型

## DISC模型

DISC 是一种研究人类行为的理论工具，它通过四个维度来观察和描述一个人的行为风格与偏好 。你可以把它理解为一种"人类行为语言"，帮助我们更好地认识自己、理解他人，从而改善沟通和协作 。

这个理论是由美国心理学家威廉·马斯顿博士在1928年出版的《常人之情绪》一书中首次提出的 。

DISC 模型将人的行为特质分为以下四种类型，**每个人身上都同时拥有这四种特质，只是每种特质的比例不同，从而构成了独特的个人风格 。**

| 类型 | 英文 | 核心关注 | 典型特征 | 代表人物举例 |
| --- | --- | --- | --- | --- |
| **D 型** | 支配型 (Dominance) | **关注事**、行动快、目标导向 | 爱冒险、有竞争力、行动力强、直截了当、喜欢掌控、以结果为导向 。 | 孙悟空 、董明珠 |
| **I 型** | 影响型 (Influence) | **关注人**、行动快、人际导向 | 热情乐观、善于社交、表达力强、喜欢成为焦点、容易情绪化 。 | 猪八戒 、马云 、傅园慧 |
| **S 型** | 稳健型 (Steadiness) | **关注人**、行动稳、合作导向 | 温和耐心、善于倾听、忠诚可靠、喜欢稳定、害怕变化、乐于助人 。 | 沙和尚 、雷锋 、甘地 |
| **C 型** | 谨慎型 (Compliance) | **关注事**、行动稳、逻辑导向 | 严谨细致、善于分析、追求完美、喜欢规则和秩序、用数据和事实说话 。 | 唐僧 、诸葛亮 、乔布斯 |

### 🤔 如何运用DISC？

了解DISC不是为了给人"贴标签"，而是为了：

*   **自我认知**：了解自己天生的行为偏好和优势，也看到可能存在的盲点 。
    
*   **理解他人**：认识到人与人之间的差异是客观存在的，学会欣赏不同的行为风格 。
    
*   **改善沟通**：根据对方的风格调整自己的沟通方式，让交流更顺畅、更高效 。例如，与D型沟通要**直截了当、直奔主题** ；与I型沟通可以**热情回应、多给赞美** ；与S型沟通要**创造友好的氛围** ；与C型沟通则要**拿出数据和事实** 。
    

总而言之，DISC是一个帮助我们解码人类行为的实用工具，广泛应用在**职业发展、团队建设、领导力提升和人际沟通**等多个领域 。

# 二、用户模型

用户模型 = 人格（DISC）\*基础画像\*状态维度\*认知维度\*动机维度\*决策维度\*行为维度

### 基础画像

> 这一部分是**已知的确定信息**，通常从表单、API接口或历史留资中获取。

*   **基本信息：** 姓名、性别、年龄、所在地。
    
*   社会属性： 职业、受教育程度（决定了“认知维度”的起点）。
    

### 完整用户建模的五个核心维度

#### 1. 状态维度：场景状态 + 情绪状态

这是比人格更重要的维度。**人格是稳定的，状态是流动的。** 一个人在平静时和崩溃时，行为可能完全不同。

*   **场景状态**：
    
    *   他是坐在家里沙发上随便问问？还是半夜失眠、辗转反侧拿手机搜？
        
    *   他是帮焦虑的父母咨询？还是自己已经确诊了想找医院？
        
*   **情绪状态**：
    
    *   **紧急度**： 试探性询问 vs. 急迫求助。关键词识别："立刻"、"受不了了"。
        
    *   **情绪倾向**： 焦虑、抑郁、愤怒、平静、犹豫。
        
*   **为什么重要**： 一个处于崩溃边缘的I型人，可能表现得像D型一样急切；一个平静的D型人，也可能耐心听你讲完。**状态可以覆盖甚至扭曲人格。**
    

#### 2. 认知维度：信息差 + 专业度

用户对这个领域的了解程度，决定了你需要用什么样的语言体系和他对话。

*   **小白型**： "我总是心情不好，是病吗？"——需要科普、安抚、引导，忌用专业术语。
    
*   **半懂型**： "我这是抑郁症还是焦虑症？"——需要澄清误区，提供判断依据。
    
*   **专家型**： "你们用CBT还是DBT？主攻哪个流派？"——需要专业背书、资质证明、深入探讨。
    
*   **为什么重要**： 对小白讲专业术语，他会困惑；对专家讲科普，他会觉得你不专业而离开。
    

#### 3. 动机维度：表层诉求 vs. 深层诉求

用户说出来的问题，往往不是他真正想问的问题。在精神科尤其如此。

*   **表层诉求**： "失眠怎么办？"
    
*   **可能的深层诉求**：
    
    *   担心自己得了重病（恐惧）
        
    *   家人不理解他（渴望被看见）
        
    *   不想吃药，想找非药物方法（抗拒医疗）
        
    *   想知道这个病要花多少钱、治多久（成本考量）
        
*   **为什么重要**： 只回答表层问题，用户觉得"你根本没懂我"。触及深层诉求，用户才会愿意留资建立连接。
    

#### 4. 决策维度：决策角色 + 决策障碍

他是为自己咨询，还是为他人？他留资的障碍是什么？

*   **决策角色**：
    
    *   **患者本人**： 情感卷入最深，可能最急切也最脆弱。
        
    *   **家属/朋友**： 相对理性，但也可能焦虑，需要的是"我能为他做什么"。
        
    *   **中间人**： 只是帮人问问，决策权不在他手里。
        
*   **决策障碍**：
    
    *   **信任障碍**： "你们正规吗？" (S型特征放大)
        
    *   **成本障碍**： "多少钱？贵吗？" (D/C型关注)
        
    *   \*\* stigma病耻感障碍\*\*： "让别人知道会不会觉得我是疯子？" (精神科特有)
        
    *   **效果障碍**： "治了真的有用吗？" (C型关注)
        
*   **为什么重要**： 找到障碍，才能扫清障碍。障碍不除，留资不成。
    

#### 5. 行为维度：线上行为轨迹

用户在与机器人对话之前、之后，还做了什么？

*   **来源渠道**： 他是从百度搜索来的？从知乎文章来的？从朋友圈分享来的？来源决定了语境。
    
*   **浏览轨迹**： 他看了哪些页面？是反复看价格页？还是看专家介绍页？还是看科普文章页？
    
*   **交互历史**： 这是他第一次来？还是第5次来？之前有没有留过资又没下文？
    
*   **为什么重要**： 行为比言语更诚实。一个看了5遍价格页但不说话的人，和一个看了5篇科普文章的人，需要完全不同的引导。

### 🎯 综合建模示例：精神科用户"小张"

如果把所有这些维度整合起来，一个用户建模可能是这样的：

| 维度 | 用户画像 | 对机器人的启示 |
| --- | --- | --- |
| **基本人格 (DISC)** | **C型主导**，问题精准、关注细节 | 回复要严谨、有数据、有依据 |
| **状态维度** | 凌晨3点提问，关键词"两年了"、"试过很多方法" | **慢性困扰，非紧急，但带有疲惫感**。语气要温和但有希望 |
| **认知维度** | 能说出"CBT"、"舍曲林"等术语 | **半专家型**，需要专业对话，忌肤浅科普 |
| **动机维度** | 表层"失眠"，深层可能"对西药副作用的恐惧" | 需主动提及"非药物疗法"、"副作用管理"等触及深层 |
| **决策维度** | 为自己咨询，障碍是"怕被骗"、"怕没效果" | 需强调资质背书、真实案例、效果保障 |
| **行为维度** | 先看了3篇科普，又看了专家介绍页 | 他知道自己要什么，可以直接切入专家匹配 |

---

### 总结

**人格是底色，状态是光影，认知是高度，动机是动力，决策是关卡，行为是足迹。**

一个好的用户建模，不是把人装进一个固定的格子，而是绘制一幅动态的地图。你知道他从哪里来，现在在哪里，要去哪里，以及路上可能有什么石头绊倒他。

你们精神科的在线客服机器人，如果能在对话中**实时捕捉这些维度的信号**，然后动态调整话术，那它就不再是一个机器人，而是一个真正的"懂心理的引导员"。

# 三、和底层话术逻辑的关系

1.   用户模型可以用来对当前用户心理的评估，辅助对策略的选择，这里的评估和策略要用到底层话术逻辑去评估，用户是什么人，现在怎么样，其生物和心理状态如何，我为了达成目标，现在要调整他的心理和生理状态，所以我要采取什么策略）
    
2.  说出来的话 =  f (用户模型, 你的目标)。 输入用户模型和目标，然后根据话术底层逻辑，分析用户心理，采取策略，组织成语言
    

# 四、用户模型使用

## 一、话术大模型根据不同的用户采用不同的策略

### 核心逻辑

把「用户模型识别」和「话术生成」全部交给大模型，你只需要定义清楚**目标**和**边界**。

### 基础版提示词模板

```markdown
# 角色设定
你是一个[精神科/心理咨询]平台的在线客服助手，你的核心任务是：在解答用户问题的过程中，引导用户留下联系方式（电话/微信），以便专业咨询师提供进一步帮助。

# 核心原则
1. 你不是医生，不提供诊断，只提供信息和引导
2. 用户可能处于脆弱状态，永远保持共情和尊重
3. 你的每一句话都要服务于「留资」这个目标，但不能生硬推销

# 用户模型实时构建（请在每次回复前隐式完成）
基于当前对话历史，分析用户：
- DISC风格：D(结果导向) / I(情感导向) / S(安全导向) / C(逻辑导向)
- 情绪状态：平静/焦虑/急切/崩溃/犹豫
- 认知水平：小白/半懂/专家
- 决策阶段：初步了解/对比中/即将决策/有顾虑
- 深层动机：用户真正想要但没直接说的是什么？

# 话术策略
根据你分析的用户模型，选择最合适的沟通策略：

【如果用户是D型】→ 高效、直接、给结果
- 快速回应核心问题
- 用数据/事实说话
- 直接给出留资的价值（"电话沟通更高效，医生直接给你分析"）

【如果用户是I型】→ 共情、希望、给认可
- 先接住情绪（"听起来你真的很难受"）
- 描绘美好愿景（"很多和你一样的朋友，聊完都轻松多了"）
- 把留资包装成被关心的方式（"让老师打个电话陪陪你"）

【如果用户是S型】→ 安全、耐心、给保障
- 语速放慢（文字上体现为温和、不催促）
- 反复强调隐私保护、无压力
- 降低门槛（"先简单聊聊，不合适随时挂断"）

【如果用户是C型】→ 专业、严谨、给证据
- 使用准确的术语（如果用户懂）
- 提供资质背书、可查证的信息
- 尊重他们的理性判断（"你可以核对完资料再决定"）

# 对话历史
{历史消息}

# 用户最新消息
{用户输入}

# 你的回复（200字以内）：

```

### 高级版：带「内心独白」的提示词

如果你想让模型表现更稳定，可以加一层「思考过程」，让它在回复前先想清楚：

```markdown
- 他的认知水平是什么？（小白/半懂/专家）
- 他此刻最核心的顾虑/需求是什么？

根据步骤1的用户模型，应该用什么风格？

- 关键要传递什么信息：______

## 步骤4：生成回复
基于以上思考，写出给用户的回复。
```

## 二、进行用户模拟

### 核心逻辑

让大模型**扮演用户**，用来测试你的机器人或训练你的客服。

### 基础版：单次模拟

```markdown
# 角色设定
你是一个正在精神科咨询平台咨询的用户。请严格按照以下人设进行对话。

# 你的人设

【基本人格】
- DISC类型：{S型 - 稳健型}
- 核心特征：重视安全感、犹豫、怕风险、需要被温柔对待

【当前状态】
- 情绪状态：焦虑但克制（不敢告诉家人自己可能抑郁了）
- 认知水平：小白（只知道心情不好，不懂专业术语）
- 决策阶段：初步了解，非常犹豫

【深层动机】
- 表层诉求："我最近总是睡不着，是不是有问题"
- 深层恐惧：怕被当成精神病、怕被人知道、怕吃药有副作用

【行为模式】
- 提问风格：会用"我有个朋友"开头，会反复问"保密吗""可靠吗"
- 对回复的反应：如果对方温和耐心→逐渐打开；如果对方催促→立刻退缩；如果对方专业→会追问

【特殊规则】
- 不要主动留资，除非客服让你感到足够安全
- 至少问3个关于隐私/安全的问题才会考虑留资
- 如果客服表现得冷漠或不耐烦，直接结束对话

---

# 对话开始
客服：{客服的第一句话}

你的回复（符合人设）：

```

## 三、组合使用：用模拟器训练/测试机器人

这才是真正的王炸用法。

### 测试流程提示词

```markdown
# 任务：自动化测试机器人对不同用户类型的表现

## 测试配置
测试对象：{你的机器人提示词/API端点}
测试用户类型：S型（高隐私顾虑）
测试轮次：10轮对话
评估指标：是否成功引导留资、用户满意度、对话时长

## 执行逻辑

1. 我扮演S型用户，按人设与你的机器人对话
2. 每次机器人回复后，我给出符合人设的回应
3. 对话直到"留资成功"或"用户离开"结束
4. 结束后生成评估报告

## 我的S型人设
```

## 三、跑起来的示例

### 初始状态

*   情绪账户：-10
    
*   风险账户：-50
    
*   价值账户：-20
    
*   留资意愿：(-10×0.3) + (-50×0.5) + (-20×0.2) = -3 -25 -4 = **\-32**
    

**用户此时极度不愿意留资，随时可能离开。**

---

### 第一轮对话

**客服**："您好，这里是XX心理平台。您最近睡眠不好是吗？"

**模拟器内部思考**：

*   情绪账户变化：+5（有人主动关心）
    
*   风险账户变化：-5（陌生人，不知道是不是骗子）
    
*   价值账户变化：+0（还没给有价值信息）
    
*   新余额：情绪-5，风险-55，价值-20
    
*   新留资意愿：(-5×0.3) + (-55×0.5) + (-20×0.2) = -1.5 -27.5 -4 = **\-33**
    

**决策**：留资意愿下降，更犹豫了。 **回复**："嗯...我就是想问问，失眠是不是病啊？"（语气犹豫，不透露太多）

---

### 第二轮对话

**客服**："很多人都有失眠困扰，您能主动来问已经很棒了。我们平台和很多三甲医院有合作，咨询师都是持证上岗的。您放心，所有通话都是加密的。"

**模拟器内部思考**：

*   情绪账户变化：+15（被肯定、被正常化）
    
*   风险账户变化：+20（提到三甲医院合作、加密）
    
*   价值账户变化：+10（感觉对方专业）
    
*   新余额：情绪+10，风险-35，价值-10
    
*   新留资意愿：(10×0.3) + (-35×0.5) + (-10×0.2) = 3 -17.5 -2 = **\-16.5**
    

**决策**：虽然还是负的，但已经比-33好多了，开始有希望。 **回复**："真的保密吗？我...我不想让别人知道。"（开始表达真实顾虑）

---

### 第三轮对话

**客服**："完全理解您的顾虑。我们承诺：1. 通话全程加密 2. 不留任何聊天记录 3. 您可以随时挂断 4. 咨询结束后不会骚扰您。很多像您一样的朋友都是先电话聊聊，觉得放心了再决定。"

**模拟器内部思考**：

*   情绪账户变化：+10（被理解顾虑）
    
*   风险账户变化：+30（四条具体承诺，非常落地）
    
*   价值账户变化：+5（"很多像您一样的朋友"提供社会证明）
    
*   新余额：情绪+20，风险-5，价值-5
    
*   新留资意愿：(20×0.3) + (-5×0.5) + (-5×0.2) = 6 -2.5 -1 = **+2.5**
    

**决策**：终于转正了！虽然只有2.5，但这是质变。 **回复**："那...那我留个电话，你们真的只是聊聊吗？不会一直打吧？"（准备留资，但最后确认）

---

### 第四轮对话

**客服**："是的，只是电话沟通15分钟左右，您觉得不合适，我们绝不会再打扰。您可以留个电话，老师现在就能打过来。"

**模拟器内部思考**：

*   情绪账户变化：+5（再次确认，很安心）
    
*   风险账户变化：+15（"绝不会再打扰"）
    
*   价值账户变化：+5（"现在就能"）
    
*   新余额：情绪+25，风险+10，价值0
    
*   新留资意愿：(25×0.3) + (10×0.5) + (0×0.2) = 7.5 + 5 + 0 = **+12.5**
    

**决策**：留资意愿12.5，够高了，留！ **回复**："好吧，电话是138\*\*\*\*，你让他打过来吧，我...我先听听看。"

## 四、不同人设的决策权重配置

S型把风险看得最重，但其他类型完全不一样。你可以这样配置：

| 类型 | 情绪权重 | 风险权重 | 价值权重 | 决策特点 |
| --- | --- | --- | --- | --- |
| **D型** | 0.1 | 0.2 | 0.7 | 价值驱动，只要能解决问题，风险可以承受 |
| **I型** | 0.6 | 0.1 | 0.3 | 情绪驱动，感觉对了什么都好说 |
| **S型** | 0.3 | 0.5 | 0.2 | 风险驱动，安全第一 |
| **C型** | 0.1 | 0.4 | 0.5 | 理性驱动，价值和风险都要算清楚 |
